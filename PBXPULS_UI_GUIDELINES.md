# PBXPULS_UI_GUIDELINES

## Purpose

Единые правила разработки интерфейса PBXPuls.

## Общие принципы

-   Интерфейс должен быть похож на профессиональную административную
    панель.
-   Не заменять существующие экраны без необходимости.
-   Новые возможности добавлять как расширение существующих.

## Компоновка экранов

1.  Верхняя панель действий.
2.  Панель фильтров.
3.  Основная таблица.
4.  Массовые операции.
5.  Preview.
6.  Журнал изменений.

## Таблицы

-   Sticky header.
-   Hover строк.
-   Выделение строки.
-   Checkbox для массовых операций.
-   Сортировка.
-   Поиск.
-   Фильтры.

## Цветовые статусы

-   Success --- зелёный.
-   Warning --- жёлтый только для реальных проблем.
-   Error --- красный.
-   Unknown --- серый.

## Preview

Всегда показывать: - до; - после; - действие; - предупреждения; - masked
secrets.

## Apply

Никогда не запускать автоматически. Всегда через Preview и подтверждение
пользователя.
